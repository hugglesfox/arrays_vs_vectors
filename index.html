Arrays and Vectors

   A tale of storing lots of data.

Act I

Arrays

   aka Static Arrays

The humble array.

Dynamically creating variables isn't a thing in a lot of programming languages but, you still want a nice way of storing lots of items? Arrays sound like the data type for you!

Arrays are primitive data types which allows one to store multiple values of the same type using a single variable. 

                  
                    int numbers[] = {0, 1, 1, 3, 4};

Wait how does it do that?

To be able to store multiple items, we need to be able to keep track of multiple locations in memory right? And you would be right.

Static typing to the rescue!

In order to solve this we can use the fact that all types in C++ have a known size at compile time in order to work out where each element is stored from a starting address by just multiplying the length of each element by a number to give you a memory offset. So 0 would get our first item, 1 the second and so on... 

                  
                    item = num * item_length + array_start

Putting it together.

The same formula can be used for allocating a block of memory for an array. We can multiply the amount of items in the array with the size of each element to work out the over all size of the array. This means the size of an array must be known at compile time. 

                  
                    array_length = item_count * item_length

The subscript operator.

Getting an element from an array or vector by using a number is so common in programming it has it's own name, the index. So the first item in an array is said to have the index of 0. In many languages we can use what's known as the subscript operator to get an element of a certain index as shown.

The same syntax can be used along with the assignment operator to assign values to indexes in the array. 

                  
                    numbers[0];  // Indexes the 0th item
                    numbers[0] = 42;  // Assigns the 0th item to 42

So why do dynamic arrays exist?

Alright we can store lots of data, we just need to know the size of everything at compile time! Well done everyone lets all go home and congratulate ourselves for a job well done.

"But wait!"

I here you say; What if we don't know how many elements the array is going to store? Like um... What if we were reading a CSV file or asking for user input? There's no way to know how big the CSV file is or how many items the user will input at compile time!?

*Thumps table*
There must be a better way!

Act II

Vectors

   aka Dynamic Contiguous Arrays

Making things more Dynamic.

So we need to create an array where we don't need to know what it's size is at compile time. Enter the vector. 

                  
                    vector<int> numbers = {0, 1, 1, 3, 4};

Well kinda...

Internally, vectors work much the same way as arrays do. By knowing the size of each item and how many items there are, it's able to allocate the correct amount of memory as well as know where each item is. 

                  
                    item = num * item_length + array_start

So what's the difference?

There is one 1 main difference between an array and a vector:

Vectors know their own size.

Lets compare resizing arrays and vectors:

Arrays

  1. Array size is calculated at compile time.
  2. We add another item to the array.
  3. The array size was calculated at compile time...
  4. The array doesn't know it got any bigger...
  5. If there wasn't enough free space, we just overwrote something...
  6. Cry...

Vectors

  1. Vector size is stored.
  2. We add another item to the vector.
  3. Increment the item count.
  4. If there isn't enough space, no problem we know where the vector starts and it's size so we can just move it to somewhere where there is enough free memory.
  5. Vectors are awesome!

A word on stacks.

Due to the nature of how vectors are stored in memory, we can only append or remove items from the end of the vector. This makes vectors very similar to what computer scientists like to call a "stack".

A nice easy way to visualise a stack is to think of them as a tower of blocks with a stick running up the center of them to make sure they are all aligned so you can only remove one block at a time from the top of the tower. If you had to remove an item from somewhere other then the end of the stack you would have to remove all the blocks above it, remove the wanted block then put all the blocks which were above it back on. Adding a block in the middle of the stack involves much the same procedure.

Removing something from a stack is known as popping it off the stack and adding something to a stack is known as pushing it into the stack. Hence why C++ uses the methods push_back() and pop_back() to manipulate vectors. 

                  
                         |
                        *|*  <-- Top item
                       **|**
                      ***|***
                     ****|****
                    *****|*****
                   ******|******
                  *******|*******  <-- Bottom item
                 =================

Accessing all the data.

So we can store everything we could possibly need (until we run out of memory) by what about accessing that data?

Currently if we wanted to print all the items in our vector then we need to manually index and print each element.

Eww...

Lets fix that. 

                  
                    cout << numbers[0];

Act III

For Loops

   aka The better while

Iterating iterables.

Repeating code for each item in an array/vector is exactly what for loops are designed for.

There are 2 main types of for loops in C++ for loops and for each loops.

For Modifies a variable until a condition is met.
For each Gets each element in an iterable (array, vector, etc).

Which variant you use depends on what your trying to achieve. 

                  
                    vector<int> numbers = {0, 1, 1, 3, 4};
                
                    // Conventional C++ for loop
                    for (int i = 0; i < numbers.size(); i++) {
                      cout << numbers[i];
                    }
                
                    // C++ for each loop
                    for (int n : numbers) {
                      cout << n;
                    }

Act VI

Conclusion

   The epic conclusion

Arrays or vectors, which should I use?

So now we know how arrays and vectors work, when should one use one or the other?

In general if you can't chose between an array and a vector, go with a vector. They are a lot easier to work with as they provide a lot more flexibility at little to no performance hit over an array. There are use cases where an array is the best tool for the job such as situations where the array doesn't need to change after compile time but those use cases are few and far between.

TL;DR: You'll know when you need an array, otherwise just use a vector.
